The concept of Island

The concept of "Island" in JavaScript frameworks refers to a modern web architecture pattern known as Islands Architecture. This approach involves rendering the majority of a web page as fast, static HTML on the server, while smaller, interactive JavaScript components—called "islands"—are selectively hydrated on the client only where interactivity or personalization is needed (e.g., an image carousel or a button).

Key Characteristics of Islands Architecture

  • Static Base with Interactive Islands: The page is mostly static HTML, with isolated interactive components ("islands") embedded within it.
  • Selective Hydration: Instead of hydrating the entire page as a single JavaScript app (like in traditional SPAs), only the individual islands are hydrated, reducing JavaScript payload and improving performance.
  • Isolation: Each island runs independently and can be built using different UI frameworks (React, Preact, Svelte, Vue, SolidJS, etc.) on the same page without interference.
  • Performance Optimization: By limiting JavaScript to only interactive parts, it reduces load times and improves responsiveness, especially for content-heavy sites with limited interactivity.

Origins and Evolution

The term "component island" was coined by Katie Sylor-Miller at Etsy in 2019, and later popularized and expanded by Jason Miller (creator of Preact) in 2020. Astro is one of the first mainstream frameworks to implement this pattern natively, enabling partial hydration and mixing multiple UI frameworks seamlessly on a single page.

Benefits

  • Faster page loads due to less JavaScript being sent to the client.
  • Improved user experience by hydrating only interactive components.
  • Flexibility to combine static and dynamic content efficiently.
  • Easier maintenance and scalability by isolating interactive widgets

The other method: HTMX

HTMX is an open-source, lightweight JavaScript library that extends HTML with custom attributes (prefixed with hx-) to enable AJAX, CSS transitions, WebSockets, and Server-Sent Events directly from HTML without writing additional JavaScript code. It allows developers to create dynamic, interactive web interfaces by defining server communication and UI updates declaratively in HTML markup.

Key Features of HTMX:

  • AJAX Requests via HTML Attributes: You can use attributes like hx-get, hx-post, hx-put, and hx-delete on any HTML element to make HTTP requests and update parts of the page dynamically without a full reload.
  • Event Triggers: The hx-trigger attribute controls when these requests happen, supporting various events such as clicks , mouseenter , or custom delays and throttling.
  • Partial Page Updates: Server responses (usually HTML fragments) can be inserted into specific parts of the DOM, enabling fine-grained updates and improving performance by avoiding full page reloads.
  • Hypermedia-Driven Approach: HTMX emphasizes using HTML as the primary data format instead of JSON, aligning with REST principles and simplifying client-server interactions.
