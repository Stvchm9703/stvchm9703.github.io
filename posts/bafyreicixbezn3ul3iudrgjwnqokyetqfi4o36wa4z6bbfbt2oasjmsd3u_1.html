[Blogger CMS - 2] Is a modern move, or just a stupid take? create the conversion tool for article

4/23/2025
By Steven,

TD;DR

" spend some time on organizing mind and target, then spend less on remaking wheel"

Intro

currently, i am creating a profile page for my career, and I need something a cool tool for my note taking, and article writing.

And, so far I pick Anytype instead of Notion, free and self-host, self management for every data, while as a web developer, it still can have flexible area for reforming the data to be content editor; how lovely it is !

Then, I have been working around 1 week for blog layout (page render), and 1 week for tool building.

A question take, or a wrong take?

at first, I was thinking about like, what if the exported file are crazy large (e.g. 100k line of sql record), and how heavy it take for a simple workspace.

So that, in that moment, I take Rust as the first choose for this act; since:

  • I used to run a batch job to data conversion and data CRUD
  • I want to make some practices for Rust

similar options can be Golang, Typescript and Python, consider the type union (enum) and flexible, Golang can be stable but not flexible , Python and Typescript can be both flexible in typing but in concern of the memory usage, I was ignoring the sensity and kept rocking for Rust;

However, it became a problemic;

In first 2 days, I took those exported Json file to generate rust struct. at first, it sound smooth

until, I met the real content blocks;

yes, those blocks are flexible and the best feature for real end-user, but inside the data struct, nah, I seen much more type kind, and start being dum;

fornorly, I found an another breakthrough: the protobuffer file, in their official github

GitHub - anyproto/any-block: Protocol describing data structures used in Anytype software

Protocol describing data structures used in Anytype software - anyproto/any-block

I was glad that, I used to make service with gRPC service before, and made some Rust app with encode and decode protobuffer before.

Then, I took a one whole day for rewriting part of code, from JSON manual typed struct to prost generated struct.

but it also hit some issues for prost:

  • it can be prost::message , but hard to be fully implement with serde::Serialize and serde::Deserialize
  • prost's well-known-type, may consider to implement with half manually: generate the struct by the prost_build, but manually add the serde::Serialize and serde::Deserialize ;
  • and, even finetune the data-struct as single flexible field, may consider as changing serialize implement;

of course, I can make the serde implement; just like :

but consider what i facing in near future,

  • how to implement the interfaces and enum, when it won't loosing the data information for enough render information;
  • how to convert the Rust interface back to Typescript interface;

Anytype, as a software development company, as a product development company,

using Electron JS and Typescript for cross-platform Application

having Golang service and protobuff for backend service and data syncing.

Any Block for content writing, are All Component they made, and they are allow to mount those data and property.

All fields and one-of message are not unnecessary data, and all fields are not wasted;

.

and

they all should be built from typescript and proto-buffer.

.

and yes, what I end up is using Javascript and Typescript as part of frontend development tool.

what it end up is, Typescript.

so, after I handout with my friends a afternoon, I sit in front of my laptop around 4 hours, from searching with google to protoc generating the typescript interface, rewrite the code manually. And it all done the conversion tool, and have an extra result for type defined for component render in later use.

even, the code generated from ts-proto , is more easy understanding, as frontend.

I mean, yeah, why it can so complex for one-of and type-union ?

what the story going to tell:

AI can be fast making anything, but it won't fix human mind error; don't rush for anything, take some time for reflashing the brain.
